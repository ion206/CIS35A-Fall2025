UML Class Diagram: Exercise 2 (Ships)

classDiagram
    direction LR

    class Ship {
        -floating : boolean
        -capacity : int
        #name : String
        -owner : String
        -departureDate : String
        +Ship(...)
        +printDetails() : void
        +move() : void
        +scheduleVoyage(...) : void
    }

    class CargoShip {
        -cargoType : String
        -loadingMethod : String
        -numberOfCranes : int
        -cargoValue : double
        +CargoShip(...)
        +printDetails() : void
    }

    class CruiseShip {
        -ticketPrice : double
        -numberOfRooms : int
        -buffetMenu : String
        -services[] : String
        +CruiseShip(...)
        +printDetails() : void
    }
    
    class ShipDriver {
        +main(String[]): void
    }

    CargoShip --|> Ship : inherits (extends)
    CruiseShip --|> Ship : inherits (extends)
    
    ShipDriver ..> Ship : uses
    ShipDriver ..> CargoShip : uses
    ShipDriver ..> CruiseShip : uses


Key OOP Concepts Shown:

Inheritance Hierarchy: Both CargoShip and CruiseShip inherit from the Ship base class.

Polymorphism: The printDetails() method is implemented in Ship and overridden in both child classes to provide specialized output.

Encapsulation: All properties are private/protected and initialized via constructors.
